# 健身记录小程序 — 版本路线图

当前版本：**v0.2.0**（2026-07-02）

---

## v0.2.0 — 体验打磨 ✅ 已完成

目标：让现有功能更好用

### 训练页
- [x] 「最近使用」区域：添加动作时优先显示最近用过的动作
- [x] 自动填充上一组的重量/次数（减少重复输入）
- [x] 训练中可修改备注
- [x] 训练完成页：显示本次训练摘要（几个动作、总组数、耗时）

### 首页
- [x] 显示具体动作名而非仅仅"已训练 N 分钟"
- [x] 下拉刷新

### 记录页
- [x] 删除训练记录
- [x] 按动作名称搜索筛选
- [x] 显示每条训练的组数

### 个人页
- [x] 训练天数统计（按日期去重）
- [x] 点击常用动作查看该动作的历史记录

### 技术升级（v0.2.0 额外完成）
- [x] 全部页面 JS → TypeScript（严格模式）
- [x] 训练保存改为云函数事务（防止数据不完整）
- [x] 统一错误处理（用户可见 toast + 指数退避重试）
- [x] 3 个自定义组件（set-row / exercise-card / stat-summary）
- [x] 3 个云函数（initDB / saveWorkout / getProfileStats）
- [x] 首页 5 分钟缓存 + 游标分页 + 骨架屏
- [x] tabBar 图标

---

## v0.3.0 — 训练模板（下一版本）

目标：不用每次手动添加动作

### 模板功能
- [ ] 保存当前训练为模板（命名 + 预设动作和组数）
- [ ] 「从模板开始」：开始训练时选择模板，一键载入动作和组数
- [ ] 模板列表页（在「我的」Tab 中）
- [ ] 编辑/删除模板

### 数据结构新增
```
templates（模板）
├── _id
├── name: string          # 模板名称，如"胸背日"
├── exercises: [{exercise_id, name, sets_count}]
├── _openid
└── created_at
```
