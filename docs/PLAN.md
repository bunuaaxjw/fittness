# 健身微信小程序 — 基础方案

## 一、项目目标

### 要开发的
一个微信小程序，帮助用户记录和管理健身训练，主要解决"不知道自己练了什么、有没有进步"的问题。

### 要学习的
通过这个项目实践 **Vibe Coding** 方法论——即以 AI 对话驱动开发，让人专注于产品决策和方向把控，AI 负责具体的代码实现。你扮演的是**产品经理 + 架构师 + 代码审查者**的角色，AI 是你的工程团队。

---

## 二、技术选型

### 方案 A：原生微信小程序（推荐 ✅）

| 维度 | 说明 |
|------|------|
| 语言 | JavaScript / TypeScript |
| 界面 | WXML（类 HTML）+ WXSS（类 CSS） |
| 工具 | 微信开发者工具（官方 IDE） |
| 后端 | 微信云开发（CloudBase），免服务器运维 |
| 数据库 | 微信云数据库（MongoDB 文档型） |
| 优势 | 官方文档齐全、调试方便、体积小、审核快 |
| 劣势 | 语法特殊（非标准 HTML/CSS），AI 训练数据相对少 |

### 方案 B：Taro 跨端框架

| 维度 | 说明 |
|------|------|
| 语言 | TypeScript + React/Vue 语法 |
| 编译 | Taro 编译为微信小程序代码 |
| 优势 | 可写标准 React，AI 更熟悉；未来可编译到 H5/支付宝等 |
| 劣势 | 多一层编译，调试链路长；部分小程序能力受限 |

> **推荐方案 A**：对于第一个小程序项目，用原生方式能让你更深入理解小程序的运行机制。遇到问题时 AI 也能直接查阅官方文档给出方案。后续如果想扩展多端，再迁移到 Taro 也不迟。

## 三、MVP 功能范围

### 第一版包含 4 个 Tab 页面：

#### 1. 首页（今天概览）
- 显示今日训练状态（未开始/进行中/已完成）
- 快速开始训练按钮
- 最近训练记录预览（最近 3 条）
- 本周训练统计（次数、时长）

#### 2. 训练（核心页面）
- 选择训练动作（从动作库）
- 记录每组：动作名、重量(kg)、次数、备注
- 添加/删除组
- 完成训练后保存

#### 3. 记录（历史）
- 按日期列表展示历史训练
- 点击可查看单次训练详情
- 按动作筛选

#### 4. 我的（个人中心）
- 个人资料（头像、昵称）
- 累计数据（总训练次数、总时长）
- 常用动作管理

### 不做在第一版：
- ❌ 训练计划/模板
- ❌ 社区/排行榜
- ❌ AI 教练建议
- ❌ 复杂图表分析
- ❌ 饮食记录

## 五、数据模型

```
exercises（动作库）         workouts（训练记录）         sets（组记录）
┌──────────────────┐      ┌───────────────────┐       ┌───────────────────┐
│ _id              │      │ _id               │       │ _id               │
│ name: string     │◄─────│ exercise_id       │       │ workout_id        │
│ body_part: enum  │      │ _openid (用户)     │◄──────│ exercise_name     │
│ category: enum   │      │ date: Date        │       │ weight_kg: number │
│ is_preset: bool  │      │ duration_min: int │       │ reps: number      │
│ icon: string     │      │ notes: string     │       │ notes: string     │
│ created_by: openid│     │ created_at: Date  │       │ sort_order: int   │
└──────────────────┘      └───────────────────┘       └───────────────────┘

body_part 枚举值：  胸 / 背 / 腿 / 肩 / 手臂 / 核心 / 全身
category 枚举值：  自由重量 / 器械 / 自重 / 有氧
```

## 七、Vibe Coding 开发流程

### 核心理念：你主导方向，AI 实现细节

```
你的角色                         AI 的角色
────────                       ──────────
• 定义需求和功能                • 生成页面代码
• 做技术选型决策                • 写工具函数
• 审查代码质量                  • 修 bug
• 提供设计方向                  • 写样式
• 决定发布时机                  • 处理边界情况
```

### 每个功能的标准开发流程：

```
1. 你说："我要实现 XX 功能"
2. AI 提问澄清细节（你确认或调整）
3. AI 生成代码
4. 你复制到微信开发者工具预览
5. 效果不对？截图/描述 → AI 修正
6. 满意 → 进入下一个功能
```

### 本项目具体推进步骤：

| 阶段 | 内容 | 产出 |
|------|------|------|
| **① 项目骨架** | 用微信开发者工具创建项目，配置云开发 | 空项目能跑 |
| **② 全局框架** | tabBar + 4 个空页面路由 + 全局样式 | 底部导航可切换 |
| **③ 数据层** | 云数据库建表 + db.js 封装 CRUD | 能读写数据 |
| **④ 动作库** | exercise-pick 页面 + 预设数据导入 | 可选动作 |
| **⑤ 训练记录** | workout + workout-detail 页面 | 核心闭环 |
| **⑥ 首页** | 今日状态 + 快速入口 + 统计 | 首页有内容 |
| **⑦ 历史+个人** | 记录列表 + 个人中心 | 完整 MVP |

每个阶段都是一次独立的 Vibe Coding 对话，目标明确、可验证。
